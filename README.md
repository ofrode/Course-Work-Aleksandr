# Низкоуровневый редактор блочного устройства

Курсовой проект на C: ncurses-редактор для работы с неразмонтированными блочными устройствами и образами дисков на уровне секторов.

## Возможности

- открытие блочного устройства или файла-образа в режиме `O_RDWR`;
- определение логического размера сектора и общей емкости устройства;
- посекторное чтение через `pread()` с несколькими попытками при ошибках;
- запись через `pwrite()` с `fdatasync()` и обязательной верификацией повторным чтением;
- распознавание MBR и GPT;
- отображение найденных разделов в боковой панели;
- просмотр содержимого сектора в `hex + ASCII`;
- навигация по LBA и редактирование байтов по шестнадцатеричным цифрам.

## Сборка

```bash
make
```

Если библиотека `ncursesw` установлена нестандартно, можно скорректировать `LDFLAGS` в `Makefile`.

## Запуск

```bash
sudo ./sector_editor /dev/sdb
sudo ./sector_editor /dev/nvme0n1 5
./sector_editor disk.img
```

Второй аргумент задает число повторных попыток чтения/записи.

## Управление

- `← → ↑ ↓` - перемещение по байтам сектора
- `PgUp / PgDn` - переход между секторами
- `g` - переход к сектору по номеру LBA
- `r` - повторное чтение текущего сектора
- `w` - запись текущего сектора на устройство
- `q` - выход
- `0-9`, `a-f`, `A-F` - редактирование байта по полубайтам

## Важные ограничения

- программу нужно запускать только на не смонтированных устройствах;
- запись в системные диски может привести к потере данных;
- для записи в `/dev/sdX`, `/dev/nvme*` обычно требуются права `root`;
- текущая версия показывает основные структуры GPT/MBR, но не реализует полный разбор всех возможных служебных форматов ФС.
